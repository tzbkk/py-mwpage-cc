# 注意事项

## 禁止删除页面

**所有脚本禁止删除 Wiki 页面。**

- 最多只能移动（重命名）页面
- 移动页面时会自动创建重定向
- 如需删除页面，请手动在 Wiki 上操作

### 原因

- 删除操作不可逆
- 可能误删重要内容
- 保留重定向可以维持页面历史和外部链接

### 相关代码

`fandom_bot.py` 中不包含 `delete` 方法，只提供：

- `edit_page()` - 编辑页面内容
- `move_page()` - 移动/重命名页面

如发现任何脚本试图删除页面，请立即报告。

## 禁止修改脚本

**转换页面时，不要修改任何脚本文件。**

### 说明

所有转换工具（如 convert_page.py、convert_category.py、convert_template.py 等）都只用于：
- 读取和修改 Wiki 页面的**内容**
- 移动/重命名 Wiki 页面
- 更新页面中的链接或分类引用

**这些工具不会也不会修改任何脚本文件或代码文件。**

### 重要提醒

1. **脚本只操作 Wiki 页面**
   - 工具通过 API 读取和编辑 Wiki 页面
   - 不会读取或修改本地文件系统中的脚本文件
   - 不会修改 `src/` 目录下的任何 `.py` 文件

2. **代码修改需要手动进行**
   - 如需修改脚本功能，请使用代码编辑器（如 VSCode、vim 等）
   - 修改脚本后需要测试再使用
   - 建议使用 Git 进行版本控制

3. **文件安全**
   - 脚本读取的 `pages.txt`、`categories.txt` 等文件只包含页面名称
   - 这些文件不会被脚本修改
   - 脚本不会读取或修改其他 `.py` 文件

### 常见误解

**误解 1**：运行脚本会修改代码文件
- **错误**：脚本不会修改任何代码文件
- **正确**：脚本只修改 Wiki 页面内容

**误解 2**：转换会改变脚本行为
- **错误**：转换不会改变脚本文件
- **正确**：脚本逻辑不会因为运行转换而改变

**误解 3**：需要备份脚本文件
- **错误**：不需要担心脚本被修改
- **正确**：但建议使用 Git 管理代码版本

### 安全保障

- 所有脚本都只调用 `fandom_bot.py` 中定义的方法
- `fandom_bot.py` 只提供页面操作 API，不提供文件修改 API
- 脚本没有读写其他脚本文件的权限
- 如发现任何脚本试图修改代码文件，请立即报告
