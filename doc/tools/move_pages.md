# move_pages.py - 页面移动工具

## 功能

批量移动/重命名页面到简体中文名称。

## 用法

```bash
python src/move_pages.py [page1] [page2] ...
python src/move_pages.py --from-file pages.txt
```

### 参数

- `page1, page2, ...`: 页面名称列表
- `--from-file FILE`: 从文件读取页面列表
- `--dry-run`: 预览模式，不实际移动

## 示例

```bash
# 指定页面
python src/move_pages.py "雖然不會說出口。" "愛歌"

# 批量处理
python src/move_pages.py "頁面1" "頁面2" "頁面3"

# 从文件读取
python src/move_pages.py --from-file pages.txt

# 预览模式
python src/move_pages.py "雖然不會說出口。" --dry-run
```

### 文件格式

pages.txt 文件格式（以 # 开头的行会被忽略）：
```
雖然不會說出口。
愛歌
小小戀歌
# 这是注释
心血來潮的浪漫
```

## 工作流程

对于每个页面：

1. 将页面名转换为简体
2. 如果名称有变化：
   - 检查页面是否存在
   - 移动页面到新名称
   - 不创建重定向
3. 如果名称无变化，跳过

## 输出示例

```

共 3 个页面

[1/3] 雖然不會說出口。
  目标名称: 虽然不会说出口。

[2/3] 愛歌
  目标名称: 爱歌
  ✓ 已移动

[3/3] STARS
  ℹ️  无需移动（名称已经是简体）

✅ 完成统计:
  - 总页面数: 3
  - 已移动: 2
  - 跳过: 1

```

## 特性

- 支持从命令行参数或文件读取页面列表
- 预览模式 (--dry-run) 查看将要执行的操作
- 只移动需要改名的页面
- 自动跳过已经是简体的页面
- 不创建重定向（避免冗余）
- 统计信息显示成功/失败/跳过数量

## 使用场景

- 批量重命名页面
- 繁体页面名转简体
- 统一页面命名规范

## 注意事项

- 移动后原页面名不再可用
- 可能影响其他页面的链接
- 移动操作不可逆
- 建议先使用 --dry-run 预览
- 建议先在小范围测试
- **脚本只移动 Wiki 页面，不会修改任何脚本文件**
